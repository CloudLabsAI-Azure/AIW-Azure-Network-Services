## Task 1: Provision Azure Firewall

In this task, you will be creating Azure Firewall. Azure Firewall is a cloud native network security service. It offers fully stateful network and application level traffic filtering for VNet resources, with built-in high availability and cloud scalability delivered as a service. You can protect your VNets by filtering outbound, inbound, spoke-to-spoke, VPN, and ExpressRoute traffic.

1. On the Azure Portal **Home** page, search for **Azure Firewall** and select it.

    ![firewall](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/firewall.png?raw=true)
    
2. Click on **Create firewall** under **Firewalls** page.

    ![create firewall](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/createfirewall.png?raw=true)
    
3. Under the **Basics** tab of **Create a firewall** page, enter the below information:

     - Subscriptio: Choose your **Subscription** from drop-down list.

     - Resource group: Select **hands-on-lab-<inject key="DeploymentID" enableCopy="false"/>** from the drop-down list

     - Name: Enter **AzFW-<inject key="DeploymentID" enableCopy="true"/>**

     - Region: Choose **East US**

     - Firewall management: **Use a Firewall Policy to manage this firewall**

       ![firewall](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/fw.png?raw=true)

     - Firewall policy: Click on **Add new(1)**, enter Policy name as **AFPolicy(2)** and Region as **East US(3)**, then click on **Yes(4)**

        ![FWP](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/FWP.png?raw=true)
        
     - Choose Virtual Network: **Use existing** and select Virtual network name as **NSVnet-<inject key="DeploymentID" enableCopy="false"/>** from drop-down list.

     - Public IP Address: Click on **Add new(1)**, enter Name as **FirewallIp(2)** and click on **Ok(3)**, then select **Review + Create**

         ![create](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/FWi.png?raw=true)

4. Review the configuration and select **Create**.

    ![create](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/ip1.png?raw=true)
    
5. After creating the firewall successfully, click on **Go to resource** .

    ![home](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/rs.png?raw=true)
    
6. Select the created Firewall from overview page of Resource Group page.

    ![rc](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/firewalll.png?raw=true)
    
7. Select **Firewall public IP** from the Overview page of **AzFW-<inject key="DeploymentID" enableCopy="false"/>**.

     ![a](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/a.png?raw=true)
     
 8. Copt the **Public Ip Address** of firewall and note it down the text editor. You will be using it in the next tasks.

     ![ip](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/ffip.png?raw=true)


## Task 2: Provision Application Gateway with WAF

In this task, you'll create an Application Gateway with WAF. Azure Application Gateway is a web traffic load balancer that enables you to manage traffic to your web applications. 


1. On the Azure Portal **Home** page, search for **Application Gateways** and select it.

     ![application](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/application.png?raw=true)

3. On the **Basics** tab of **Create application gateway** page, enter the follow the below instructions:

      - Subscription: Select your **Subscription** from frop-down list

      - Region: **East US**

      - Tier: **WAF V2**

      - Virtual network: Select **NSVnet-<inject key="DeploymentID" enableCopy="false"/>**

## Task 3: Publish Application via Azure Firewall & Application Gateway.             

In this task, you'll publish an application via Azure Firewall and Application Gateway.

1. On the Azure Portal **Home** page, search for **Azure Firewall** and select it.

    ![firewall](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/firewall.png?raw=true)
    
2.  Click on the firewall **AzFW-<inject key="DeploymentID" enableCopy="true"/>**.

     ![firewall](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/firewallc.png?raw=true)
     
3. Select **Firewall Manager** from **Settings** and click on **Visit Azure Firewall Manager to configure and manage this firewall**

    ![FM](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/fm1.png?raw=true)
    
4. Select **Azure Firewall policy** under **Firewall Manager** page and click on Firewall Policy **AFPolicy**.

    ![policy](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/AFP.png?raw=true)
   
5. Select **DNAT Rules** from **Settings** tab under **Firewall Policy** page and select **Add a rule collection**

    ![rule](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/dnat1.png?raw=true)
    
6. Under **Add a rule collection** page, enter the below details:

      - Name: **Rulecollection**

      - Rule Collection type: **DNAT**

      - Priority: **100**

      - Rule collection group: **DefaultDnatRuleCollectionGroup**

      - Under **Rules** mention the below details:  

           - Name: **DNAT**

           - Source type: Select **IP Address** from the drop-down list

           - Source: Enter *

           - Protocol: Select **TCP** from the drop-down list

           - Destination Ports: **80**

           - Destination type: Select **IP Address** from the drop-down list

           - Destination : Enter the IP address of **Firewall** that you copied in the task-1

           - Translated address: Enter the IP address of **Application gateway** that you copied in the task-2

           - Translated port: **80**

      - Click on **Add**

          ![rule](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/rulecollection.png?raw=true)
          
 7. Now copy the Public IP address of the **Application Gateway** from the text editor and browse the public ip in a new tab or different browser. You can see the web page as mentioned in the below screenshot

      ![ss](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/ipvm.png?raw=true)
      
8. Now copy the Public IP address of the **Firewall** from the text editor and browse the public ip in a new tab or different browser. You can see the web page as mentioned in the below screenshot

      ![ss](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/firewallip.png?raw=true)
       
9. This will confirm that you have published the application via Azure Firewall & Application Gateway.



## Task 4: Test Application for security and perform sample attack(XSS)

In this task you will be testing you application for security and perform sample attack like XSS. Cross-Site Scripting (XSS) attacks are a type of injection, in which malicious scripts are injected into otherwise benign and trusted websites. XSS attacks occur when an attacker uses a web application to send malicious code, generally in the form of a browser side script, to a different end user.

1. Attackers can attack your application by passing this **?q=<script>** value at the end of web application URL or IP address.
    
2. Now pass the the value **?q=<script>** at the end of your **Application gateway** and try browsing it. You can still see the webpage.
    
    ![ss](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/web.png?raw=true)
    
3. To prevent the application from attackers, Select **Appgateway-<inject key="DeploymentID" enableCopy="true"/>** from the overview page of resource group.
     
    ![rp](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/appgatew.png?raw=true)
    
 4. Under the **Application gateway** page, follow the below instructions:
    
       - Select **Web application firewall** under **Settings**
    
       - Under **Configure** balde, set Firewall Status as **Enabled** and Firewall mode as **Prevention**.
    
          ![config](https://github.com/CloudLabsAI-Azure/AIW-Azure-Network-Solutions/blob/main/media/config.png?raw=true)

       - Now move to **Rules** blade
    
       - Advanced rule configuration: **Enabled**
    
       - Select **Save**
    
         ![save](https://github.com/CloudLabsAI-Azure/AIW-Azure-Network-Solutions/blob/main/media/enable.png?raw=true)
    
 5. Now navigate back to tab where you browsed the IP Address and refresh the page. You can observe the Server Error.

## Task 5: Configure UDR and outbound traffic to go via  Azure Firewall
    
1. On the Azure Portal **Home** page, search for **Route Tables** and select it.
    
    ![route](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/route.png?raw=true)
    
2. On the **Route Tables** page, select **+ Create**.
    
    ![create](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/RTC.png?raw=true)
    
3.  Under the **Basics** tab of **Create route table**, follow the below instructions:
    
      - Subscription: Select your **Subscription**
    
      - Resource group: Select your **Resource group**
    
      - Region: Select **East US**
    
      - Name: Enter **RouteTables-<inject key="DeploymentID" enableCopy="true"/>**
    
      - Propogate gateway routes: **Yes**
    
      - Select **Review + Create**
    
         ![create](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/rbas.png?raw=true)
    
4. Review the configuration and select **Create**
    
    ![create](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/qq.png?raw=true)
    
5. After the successful deployment, click on **Go to resource**
    
    ![create](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/rtgr.png?raw=true)
    
6. Select **Routes** under **Settings** from **Route Tables** page.
    
    ![rt](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/routes.png?raw=true)
    
7.Click on **Add** from **Routes** page.
    
   ![page](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/addroute.png?raw=true)
    
8. Under **Add Route** page, enter the below information:
    
    - Route Name: Enter **NewRoute**
    
    - Address Prefix: Enter **0.0.0.0/0**
    
    - Next hop type: Select **Virtual appliance** from drop-down list
    
    - Next hop address: Enter the IP Address of Firewall.
    
    - Select **Ok**
    
      ![ok](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/routeadd.png?raw=true)
    
9. Now select **Subnets(1)** and Click on **Associate(2)**.
    
    - Under Associate subnets, enter the following details:
    
         - Virtual Network : Select **NSVnet-<inject key="DeploymentID" enableCopy="false"/>** from drop-down list.
    
         - Subnet : Select **Default** from the drop-down list.
    
         - Click on **Ok**.
    
      ![ok](https://github.com/Divyasri199/AIW-Azure-Network-Solutions/blob/prod/media/subnets.png?raw=true)
